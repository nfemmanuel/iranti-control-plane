# CP-T056 — Temporal History: Point-in-Time `asOf` Query

**Type:** Feature
**Priority:** P3
**Assigned:** frontend_developer
**Phase:** 3 (Wave 4+)
**Status:** Open — issued 2026-03-21 from cross-repo audit (L-tier, API capability exposure)
**Epic:** CP-E002 (Memory Explorer)

## Summary

The Iranti `GET /kb/query/:entityType/:entityId/:key` endpoint supports an `?asOf=<ISO timestamp>` parameter that returns the fact state at a specific point in time (the entry whose `validFrom` ≤ `asOf` < `validUntil`). The Temporal History view (CP-T036) currently shows the full interval list but does not allow operators to query a specific point in time.

This ticket adds a date/time picker to the Temporal History view that, when selected, re-queries using `asOf` and highlights the interval that was active at that moment.

## Background

From cross-repo audit 2026-03-21:
- `GET /kb/query/:entityType/:entityId/:key?asOf=ISO&includeExpired=bool&includeContested=bool` — already supported by Iranti v0.2.12
- The control plane's Temporal History view reads `GET /kb/history/:entityType/:entityId/:key` which returns all intervals. It does not currently use the `asOf` parameter on the query endpoint.
- Use case: "What did Iranti believe about this fact on March 15?" — operators debugging why an agent had stale information at a specific time.

## Acceptance Criteria

### AC-1 — `asOf` date/time picker in Temporal History view

In the Temporal History view (the per-key interval timeline at `/memory/:entityType/:entityId/:key`):
- Add a "Point in Time" date+time picker input in the view header
- When a date/time is selected: call `GET /kb/query/:entityType/:entityId/:key?asOf=<ISO>&includeExpired=true`
- If a result is returned: highlight the matching interval in the timeline (e.g., elevated border or background) and show the fact value at that time in a callout
- If no result is returned (fact did not exist at that time): show "No fact existed at this time" in the callout
- If the date picker is cleared: return to the normal full-history view with no highlight

### AC-2 — Display the returned value

When an `asOf` result is returned, show:
- The `valueRaw` (or `valueSummary`) at that point in time
- The `confidence` at that point
- The `source` and `createdBy` at that point
- The interval it falls within (with `validFrom` / `validUntil` dates)

### AC-3 — No backend changes needed

This is a pure frontend change. The `asOf` parameter is already supported by the Iranti API. The control plane backend passes query parameters through to Iranti on the `/kb/query` route — no new backend endpoint is needed.

### AC-4 — Consistent with Terminals palette

The highlight style for the active interval and the callout box must be consistent with the existing visual system (emerald accent, dark canvas).

## Implementation Notes

- The Temporal History view is in `src/client/src/components/memory/MemoryExplorer.tsx` (temporal section) or a dedicated `TemporalHistory.tsx` component
- The control plane already proxies `GET /kb/query/:entityType/:entityId/:key` — check that query params (including `asOf`) are forwarded to Iranti
- The interval list already returned by `GET /kb/history/:entityType/:entityId/:key` can be used to pre-compute which interval the `asOf` date falls within — no second request strictly required if the interval data includes `validFrom`/`validUntil`
