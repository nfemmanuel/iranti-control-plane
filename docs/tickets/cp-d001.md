# CP-D001 — P0 Defect: SQL Column Name Mismatch (snake_case vs camelCase Prisma Schema)

## ID

CP-D001

## Title

P0 Defect: All KB/Archive/Entity data routes fail against live Iranti DB — SQL uses snake_case column names, Prisma schema uses camelCase

## Type

Defect (P0 Blocker)

## Phase

1 (post-v0.1.0 acceptance — discovered by QA seed test 2026-03-20)

## Assigned

backend_developer

## Status

**OPEN — P0 BLOCKER — must fix before design partner handoff**

Discovered: 2026-03-20 by qa_engineer during CP-T030 seed test execution.

P0 — no data read paths work against the live Iranti database. Memory Explorer, Archive Explorer, Entity Detail, Temporal History, Staff Activity Stream all fail on a real DB instance.

## Problem Statement

The SQL queries in `src/server/routes/control-plane/kb.ts` use snake_case column names (`summary`, `value_raw`, `entity_type`, `created_at`, `archived_at`, `agent_id`, `valid_from`, `valid_until`, `archived_reason`, `superseded_by`, `resolution_state`, `conflict_log`, `from_entity_type`, `from_entity_id`, `to_entity_type`, `to_entity_id`, `relationship_type`) but the Iranti Postgres database (generated by Prisma) uses **camelCase column names** (`"valueSummary"`, `"valueRaw"`, `"entityType"`, `"entityId"`, `"createdAt"`, `"archivedAt"`, `"agentId"`, `"validFrom"`, `"validUntil"`, `"archivedReason"`, `"supersededBy"`, `"resolutionState"`, `"conflictLog"`, `"fromEntityType"`, `"fromEntityId"`, `"toEntityType"`, `"toEntityId"`, `"relationshipType"`).

The serializer functions (`serializeKBRow`, `serializeArchiveRow`, `serializeRelationshipRow`) already handle both cases via `row.entity_type ?? row.entityType` dual-read fallbacks — this was correctly written. But the SQL WHERE clauses and ORDER BY clauses use the snake_case names directly, causing PostgreSQL error `42703: column "summary" does not exist` before any row data reaches the serializers.

The unit tests (104/104 passing) test pure function logic and do not execute against a live database — they cannot catch this class of error.

---

## Affected Files and Specific Fixes Required

### File 1: `src/server/routes/control-plane/kb.ts`

**History endpoint explicit SELECT columns** (lines ~432–466):
Replace all snake_case column names in the explicit SELECT with quoted camelCase:

```sql
-- BEFORE (broken)
summary AS "valueSummary", value_raw AS "valueRaw", agent_id AS "agentId",
valid_from AS "validFrom", valid_until AS "validUntil", created_at AS "createdAt",
archived_at AS "archivedAt", archived_reason AS "archivedReason",
superseded_by::text AS "supersededBy", resolution_state AS "resolutionState",
conflict_log AS "conflictLog", entity_type, entity_id

-- AFTER (fixed)
"valueSummary", "valueRaw", "agentId", "validFrom", "validUntil", "createdAt",
"archivedAt", "archivedReason", "supersededBy"::text, "resolutionState",
"conflictLog", "entityType", "entityId"
```

**ORDER BY and WHERE in SELECT * queries**:
- `/kb`: `ORDER BY created_at DESC` → `ORDER BY "createdAt" DESC`
- `/archive`: `ORDER BY created_at DESC` → `ORDER BY "createdAt" DESC`
- `/entities/:entityType/:entityId`:
  - `WHERE entity_type = $1 AND entity_id = $2 ORDER BY created_at DESC` → use `"entityType"`, `"entityId"`, `"createdAt"`
  - Archive: `ORDER BY valid_from DESC` → `"validFrom" DESC`
  - Relationships: `WHERE (from_entity_type = $1 AND from_entity_id = $2) OR (to_entity_type = $1 AND to_entity_id = $2) ORDER BY created_at DESC` → all camelCase with quotes

**`buildKBWhereClause` snake_case column refs** (lines ~156–200):
```
entity_id → "entityId"
summary → "valueSummary"
value_raw → "valueRaw"
entity_type → "entityType"
key → "key"  (key is unchanged — it IS the column name)
source → "source"  (unchanged)
agent_id → "agentId"
valid_until → "validUntil"
```

**`buildArchiveWhereClause` snake_case column refs** (lines ~202–258):
```
entity_id → "entityId"
summary → "valueSummary"
value_raw → "valueRaw"
entity_type → "entityType"
agent_id → "agentId"
archived_reason → "archivedReason"
resolution_state → "resolutionState"
superseded_by → "supersededBy"
archived_at → "archivedAt"
```

**Relationships route WHERE clause** (lines ~616+):
```
from_entity_type → "fromEntityType"
from_entity_id → "fromEntityId"
to_entity_type → "toEntityType"
to_entity_id → "toEntityId"
relationship_type → "relationshipType"
```
And ORDER BY: `created_at DESC` → `"createdAt" DESC`

The QA report notes `installed_version` column error in the health check. Read health.ts and apply the same camelCase fix to any snake_case column references.

### File 3: `src/server/routes/control-plane/events.ts` (if affected)
