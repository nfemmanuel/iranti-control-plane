## Problem Statement

The Memory Explorer has two routes that render `PlaceholderView` stubs instead of real views:

- `/memory/:entityType/:entityId` → renders "Entity Detail — coming soon"
- `/memory/:entityType/:entityId/:key` → renders "Temporal History — coming soon"

Both routes are linked from the Memory Explorer via "View Related Entities →" and "View History" buttons in the expanded row detail. Users who click these buttons land on a placeholder screen with no useful information.

This is a critical product gap for v0.1.0:
- The PRD (FR1, FR2) explicitly requires entity detail pages and temporal history timelines as core Memory Explorer capabilities.
- The backend endpoints are fully implemented and tested (CP-T010, CP-T030 history endpoint).
- "Dev (Power User)" — Iranti's most technically demanding persona — has replacing saved SQL queries with the temporal history view as his primary Phase 1 job. The placeholders make the control plane fail his primary use case.

The Phase 1 retrospective identified this as the most critical Phase 1 gap. It must be closed before design partner outreach.

## Scope

**In:**

### Entity Detail View (`/memory/:entityType/:entityId`)

A dedicated page for a single entity (entityType + entityId pair) showing:

1. **Entity header**: entityType, entityId, fact count, last updated timestamp
2. **Current facts table**: all current knowledge_base facts for this entity, with key, valueSummary, confidence, source, agentId, validFrom, updatedAt — sortable by key, confidence, updatedAt
3. **Archived facts table**: all archive entries for this entity — same columns plus archivedReason, archivedAt. Collapsible or in a tab.
4. **Relationships list**: flat list of entity_relationships for this entity (entityType/entityId pairs it relates to, with relationshipType and confidence). Links to those entities.
5. **Back button**: breadcrumb back to Memory Explorer with the current filters preserved.

Navigation to temporal history: clicking a fact's key in the current facts table navigates to `/memory/:entityType/:entityId/:key`.

**Out of scope:**
- Graph visualization (CP-T032 — dependent on this view existing first)
- Inline fact editing (read-only per Phase 1 principle)

### Temporal History View (`/memory/:entityType/:entityId/:key`)

A timeline for a single fact (entityType + entityId + key):

1. **Page header**: entityType/entityId / key, breadcrumb back to entity detail
2. **Timeline list**: list of history intervals, newest first
   - Each interval: valueSummary, confidence, source, agentId, validFrom, validUntil, archivedReason (human-readable label from backend), supersededBy (if present), a "current" badge if it is the live fact
   - Clicking an interval expands to show full raw JSON (valueRaw)
3. **Empty state**: "No history — this fact has not been superseded or archived" when `hasHistory: false`
4. **Loading state**: Spinner while fetching
5. **Error state**: Unable to load history message with link to Health
