## Problem Statement

The PRD lists `entity_aliases` as a data source for Memory Explorer. CP-T002 explicitly deferred it as open question #4 because the table's shape and query patterns were unknown. Confirmed via Iranti schema investigation: the running Iranti database has only 3 tables (`knowledge_base`, `archive`, `entity_relationships`) — no `entity_aliases` table exists. Before the Memory Explorer can display alias data, two things must be determined: (1) whether entity aliases are stored somewhere in the existing schema by convention, and (2) what table shape and API endpoint design is needed to expose them.

## User Value

Operators can see aliases for entities alongside their current facts — understanding that "ticket/cp_t001" and "CP-T001" refer to the same thing, or that an entity has multiple known identifiers. Without this, the Memory Explorer may present data that appears duplicated or unresolvable.

## Scope

**In:**
- Investigate whether entity aliases are currently stored in the Iranti codebase: check `knowledge_base` conventions, check Iranti SDK source code, check any migration history
- If no alias mechanism exists: propose a `entity_aliases` table schema (columns, indexes, constraints) as a new migration
- If aliases are stored by convention in `knowledge_base` (e.g., `key="_alias"` entries): document the convention and design a query to surface them
- Produce an amendment to CP-T002: add an entity aliases endpoint to the API spec (or document how aliases are retrieved via existing endpoints)
- Flag any upstream Iranti SDK changes required to write aliases via the MCP/API surface

**Out:**
- Implementation of the entity_aliases table or migration
- Implementation of the alias API endpoint (that follows in a separate Phase 1 task once this spike is accepted)
- UI implementation

## Dependencies

- CP-T002 (API spec — this spike produces an amendment to it)
- PM decision OQ-4: new ticket CP-T006 created to resolve alias gap

## Acceptance Criteria

- [ ] Investigation documents whether entity_aliases currently exist in Iranti (as a table, a convention, or not at all). Source of determination: actual Iranti codebase inspection, not speculation.
