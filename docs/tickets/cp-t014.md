## Problem Statement

No UI exists to watch Staff activity. Operators have no way to see whether the Librarian is writing facts, whether conflicts are occurring, or whether the Archivist is decaying entries — except by querying the database manually. The Staff Activity Stream is one of the most differentiated capabilities of the control plane: making Iranti's internal behavior legible in real time.

## User Value

Operators can watch live Librarian writes, Archivist archival events, and (in Phase 2) all four Staff components in real time. They can pause the stream to inspect events, filter by component or entity, and reconnect automatically on disconnect. This closes the "is Iranti doing anything?" gap without requiring SQL or log file access.

## Scope

**In:**
- Full-page Activity Stream view (`/activity` route) per CP-T004 wireframe (Section 1.4)
- Live SSE event feed connecting to `GET /api/control-plane/events/stream`
- Newest-at-top event list with auto-scroll
- Event row display: timestamp, Staff component (color-coded per Option B Staff palette), actionType, agentId, entityType/entityId/key when present, reason when present
- Pause/resume toggle: paused state buffers new events client-side, shows buffered count badge, resume replays buffer into view
- Level filter: audit / debug toggle (audit default)
- Component filter: Librarian, Attendant, Archivist, Resolutionist checkboxes
- Agent filter: free-text input to filter by agentId
- Entity filter: free-text to filter by entityType or entityId
- "Live" mode (SSE) vs "Tail" mode (loads last N events from `/api/control-plane/events` — no live connection)
- Custom `useEventStream` hook: manages EventSource lifecycle, reconnect on disconnect, pause buffer
- Empty state for no events (new installation, no agent activity yet)
- Error state for SSE connection failure with reconnect button
- Auto-reconnect on SSE disconnect (EventSource API handles this natively with Last-Event-ID)
- Staff component color coding per Option B palette: Librarian `#F59E0B`, Attendant `#A78BFA`, Archivist `#38BDF8`, Resolutionist `#10B981`

**Out:**
- Event replay, export to file
- Deep historical filtering beyond what `/api/control-plane/events` query endpoint supports
- Resolutionist and Attendant events (not available in Phase 1 due to adapter layer limitation — show informative notice in UI)

**PM note:** Phase 1 adapter layer only emits Librarian write events and Archivist archive events. Attendant session events and Resolutionist decision events are absent. The UI must communicate this clearly — display a persistent notice: "Attendant and Resolutionist events will appear after Phase 2 native emitter upgrade." This is not a bug state, it is a documented Phase 1 limitation.

## Acceptance Criteria

- [ ] Stream connects to SSE endpoint and renders new events within 3 seconds of server emission.
- [ ] Each event row displays: timestamp (formatted readable, e.g. "10:34:12"), Staff component label (color-coded accent dot per Option B Staff palette), actionType, agentId, entityType+entityId+key when present, reason when present.
- [ ] Component filter (Librarian/Attendant/Archivist/Resolutionist checkboxes) updates the visible event set. When component filter changes, a new SSE connection is opened with updated `staffComponent` query param.
- [ ] Level filter (audit / debug) updates the visible event set via SSE reconnect with updated `level` param.
- [ ] Agent filter (free text) and entity filter (free text) work as client-side filters on the buffered event list (not SSE reconnect) to avoid excessive reconnects on typing.
- [ ] Pause toggle: paused state freezes auto-scroll and buffers incoming SSE events with a visible count badge (e.g., "47 new events"). Resume button replays all buffered events into the feed and resumes auto-scroll.
- [ ] "Live" vs "Tail" mode switch: Live = SSE connection. Tail = one-time load of last 500 events from `/api/control-plane/events`, no live connection. Mode persists within the session.
- [ ] Auto-reconnect works: when SSE connection drops, browser EventSource API reconnects automatically using `Last-Event-ID`. No user action required. A "Reconnecting..." indicator shows during the gap.
- [ ] Empty state: when no events exist in the DB, shows a meaningful message ("No Staff events yet. Start an agent session to see activity here.").
- [ ] Connection error state: if SSE endpoint returns an error event or the connection fails permanently after retries, show error state with manual "Reconnect" button.
- [ ] Phase 1 limitation notice visible: a persistent informational banner or empty-section note explains that Attendant and Resolutionist events are not yet available.
- [ ] Both light and dark mode render correctly per Option B tokens.

## Risks

- EventSource reconnect sends Last-Event-ID header — server must support cursor resume (covered in CP-T012 AC). If server does not support it, events will be duplicated on reconnect.
- Pause buffer can grow large in a high-activity session. Cap buffer at 1000 events; oldest events are dropped silently when cap is reached. Show "some events may have been dropped" notice if cap hit.
- SSE connections from two open browser tabs simultaneously: both will work independently (read-only). No cross-tab state needed.
