## Summary

This PR introduces a structured, opt-in event emitter interface (`IStaffEventEmitter`) into the Iranti core package. It instruments all four Staff components — Librarian, Attendant, Archivist, and Resolutionist — to emit discrete, typed events at every meaningful decision point: writes created, conflicts escalated, entries archived, resolutions filed, and so on. The default implementation is a zero-overhead no-op, so the change is fully backward-compatible with all existing deployments. Operators who have built, or intend to build, an observability layer (such as the Iranti Control Plane) can supply a concrete emitter implementation — for example, one that writes to a `staff_events` database table and issues a `pg_notify` broadcast — by passing it through the `IrantiConfig` object at SDK construction time. No Staff component behavior changes. No new return values. No new error paths. The emitter is a pure, non-blocking side-channel.

---

## Use Case: Iranti Control Plane Live Tail

The Iranti Control Plane is an operator surface that provides a real-time activity stream of all Staff component actions. Without native emission from the Staff components themselves, the control plane can only infer activity by polling the `knowledge_base` table for new writes and the `archive` table for archived entries — a strategy that cannot recover Attendant events (handshake, attend, reconvene), Resolutionist events (resolution filed, escalation deferred), or negative-outcome Librarian events (`write_rejected`, `write_deduplicated`).

With this PR, the control plane's backend supplies a `DbStaffEventEmitter` that writes each event as a row to a `staff_events` table, then calls `pg_notify('staff_events_channel', eventId)`. A dedicated PostgreSQL LISTEN client in the control plane server receives the notification, fetches the row by `eventId`, and broadcasts it to all open SSE connections. End-to-end latency from Staff action to browser is expected to be 10–50ms on local hardware, compared to a theoretical maximum of 1–2 seconds with polling.

The control plane is the immediate consumer, but the interface is general enough for any observability integration: log forwarders, metrics exporters, OpenTelemetry bridges.

## Changes

### New files

#### `src/lib/staffEventEmitter.ts`

Defines the public contract for the emitter system. Contains:

- `StaffComponent` — `'Librarian' | 'Attendant' | 'Archivist' | 'Resolutionist'`
- `EventLevel` — `'audit' | 'debug'`
- `StaffEvent` — the full event structure (eventId, timestamp, staffComponent, actionType, agentId, source, entityType, entityId, key, reason, level, metadata, emittedAt)
- `StaffEventInput` — what callers pass to `emit()` (omits eventId, timestamp, emittedAt — filled by the emitter)
- `IStaffEventEmitter` — the interface all Staff components depend on
- `NoopEventEmitter` — the default implementation; `emit()` is an empty function body
- `buildStaffEvent(input)` — utility that fills in eventId (UUID v4), timestamp, and emittedAt from a `StaffEventInput`

```typescript
// src/lib/staffEventEmitter.ts

import { randomUUID } from 'crypto';

export type StaffComponent =
  | 'Librarian'
  | 'Attendant'
  | 'Archivist'
  | 'Resolutionist';

export type EventLevel = 'audit' | 'debug';

export interface StaffEvent {
  eventId: string;           // UUID v4 — generated by emitter
  timestamp: string;         // ISO 8601 — generated by emitter
  staffComponent: StaffComponent;
  actionType: string;
  agentId: string;
  source: string;            // 'mcp' | 'api' | 'claude_code' | 'cli' | 'internal'
  entityType?: string | null;
  entityId?: string | null;
  key?: string | null;
  reason?: string | null;
  level: EventLevel;
  metadata?: Record<string, unknown> | null;
  emittedAt: string;         // ISO 8601 — set at emit() call site
}

export type StaffEventInput = Omit<StaffEvent, 'eventId' | 'timestamp' | 'emittedAt'>;

export interface IStaffEventEmitter {
  /**
   * Emit a Staff event. Implementations MUST be synchronous from the caller's
   * perspective: emit() returns void immediately. Async delivery (DB write,
   * network I/O) is handled internally. Implementations MUST NOT throw —
   * all errors are caught internally and logged.
   */
  emit(event: StaffEventInput): void;
}

export class NoopEventEmitter implements IStaffEventEmitter {
  emit(_event: StaffEventInput): void {
    // Intentionally empty. Zero overhead for deployments without an
    // observability integration. This is the default.
  }
}

export function buildStaffEvent(input: StaffEventInput): StaffEvent {
  const now = new Date().toISOString();
  return {
    ...input,
    eventId: randomUUID(),
    timestamp: now,
    emittedAt: now,
  };
}
```

#### `src/lib/staffEventRegistry.ts`

Module-level singleton registry. Staff components import `getStaffEventEmitter()` and call it at each injection point. The SDK sets the active emitter once at construction. No threading through function signatures required.

```typescript
// src/lib/staffEventRegistry.ts

import { IStaffEventEmitter, NoopEventEmitter } from './staffEventEmitter';

let _emitter: IStaffEventEmitter = new NoopEventEmitter();

/**
 * Set the active Staff event emitter. Called once by the Iranti SDK constructor
 * when a concrete emitter is provided. Must not be called after startup in
 * production — use resetStaffEventEmitter() in tests only.
 */
export function setStaffEventEmitter(emitter: IStaffEventEmitter): void {
  _emitter = emitter;
}

/**
 * Get the currently active emitter. Returns the NoopEventEmitter if no
 * concrete emitter has been set.
 */
export function getStaffEventEmitter(): IStaffEventEmitter {
  return _emitter;
}

/**
 * Reset to NoopEventEmitter. For use in test beforeEach/afterEach only.
 */
export function resetStaffEventEmitter(): void {
  _emitter = new NoopEventEmitter();
}
```

### Modified files

#### `src/librarian/index.ts`

Adds `getStaffEventEmitter().emit(...)` calls at five decision return sites within `librarianWrite` and its internal helpers. No changes to function signatures.

- Before `return { action: 'created', ... }` — emits `write_created` (level: `audit`)
- Before `return { action: 'updated', ... }` inside `resolveConflict` — emits `write_replaced` (level: `audit`)
- After `escalateConflict()` completes — emits `write_escalated` (level: `audit`)
- Before each `return { action: 'rejected', ... }` (5+ sites for different rejection reasons) — emits `write_rejected` (level: `audit`)
- On idempotent replay path when `requestId` receipt is found — emits `write_deduplicated` (level: `debug`)

#### `src/attendant/AttendantInstance.ts`

Adds `getStaffEventEmitter().emit(...)` calls at the end of each public method:

- End of `handshake()` — emits `handshake_completed` (level: `debug`)
- End of `reconvene()` — emits `reconvene_completed` (level: `audit`)
- End of `attend()` — emits `attend_completed` (level: `debug`)
- End of `observe()` — emits `observe_completed` (level: `debug`)
- In `onContextLow()` or session eviction path — emits `session_expired` (level: `audit`)

Note: `AttendantInstance` does not receive the originating source (MCP vs CLI vs API) as a call parameter in the current implementation. Attendant events use `source: 'internal'` in this PR. Threading source through the call chain is a follow-up improvement.

#### `src/archivist/index.ts`

Adds `getStaffEventEmitter().emit(...)` calls in the Archivist's maintenance functions:

- In `archiveExpired()`, after each `archiveEntry()` succeeds — emits `entry_archived` (level: `audit`)
- In `applyMemoryDecay()`, after each decay-triggered `archiveEntry()` — emits `entry_decayed` (level: `audit`)
- In `processEscalationFile()`, after escalation file is read and resolution is recorded — emits `escalation_processed` (level: `audit`)
- In `processEscalationFile()`, after archive row `resolutionState` is set to resolved — emits `resolution_consumed` (level: `audit`)
- After `runArchivist()` loop completes — emits `archive_scan_completed` with ArchivistReport counts (level: `debug`)

#### `src/resolutionist/index.ts`

Adds `getStaffEventEmitter().emit(...)` calls at resolution outcome points:

- After `writeFile(filePath, resolvedContent)` succeeds — emits `resolution_filed` (level: `audit`)
- Before `skipped++` when operator skips an escalation — emits `escalation_deferred` (level: `audit`)

#### `src/sdk/index.ts`

- Adds `staffEventEmitter?: IStaffEventEmitter` to `IrantiConfig`.
- In the `Iranti` constructor: `setStaffEventEmitter(config.staffEventEmitter ?? new NoopEventEmitter())`.
- Re-exports `IStaffEventEmitter`, `NoopEventEmitter`, `setStaffEventEmitter`, `resetStaffEventEmitter` for public consumers.

## Testing

### Unit test strategy (one file per Staff component)

Each test file uses a mock emitter that records all `emit()` calls:

```typescript
class RecordingEmitter implements IStaffEventEmitter {
  readonly calls: StaffEventInput[] = [];
  emit(event: StaffEventInput): void { this.calls.push(event); }
}
```

**Pattern for each test:**
1. Instantiate a `RecordingEmitter`.
2. Call `setStaffEventEmitter(recorder)` before the action.
3. Trigger the Staff component action under test (mocking the DB layer as needed).
4. Assert `recorder.calls.length === expectedCount`.
5. Assert `recorder.calls[0].actionType === 'write_created'` (etc.).
6. Assert key metadata fields are present and correct.
7. Call `resetStaffEventEmitter()` in `afterEach`.

Tests do not require a live database. All DB interactions are mocked (Jest mocks or `pg-mem`).

## Rollout Safety

The no-op default guarantees this PR merges with zero impact on any existing deployment.

- **Existing SDK users**: if `config.staffEventEmitter` is not provided (which is true for all current callers), `setStaffEventEmitter(new NoopEventEmitter())` is called. `emit()` is called at each injection point and returns immediately with a single empty function body — effectively free at V8 JIT steady state.
- **No function signature changes** to `librarianWrite`, `runArchivist`, or `resolveInteractive` — these are the three public functions called by the SDK and API routes. They remain identical.
- **No new dependencies** in the Iranti package itself — `IStaffEventEmitter` is a local interface. The concrete `DbStaffEventEmitter` that writes to PostgreSQL lives in the consumer (the control plane), not in the upstream package.
- **No schema changes** to any Iranti-managed table. The `staff_events` table is created by the control plane's migration, not by the upstream package.
- **No behavioral changes** to conflict resolution, archiving, or any other Staff operation.

---

## Rollback

If the emitter causes unexpected issues in production:

1. Remove the `staffEventEmitter` field from the `IrantiConfig` passed to `new Iranti(...)` — or set it to `new NoopEventEmitter()`.
2. Restart the Iranti server.
3. Result: all injection points call `NoopEventEmitter.emit()`, which is a no-op. Zero impact on Staff operations.

There is no database migration to reverse — the `staff_events` table is managed by the control plane, not the upstream package. The upstream package has no dependency on that table existing.

## Notes and Known Limitations

1. **`source` field in Attendant events**: The `AttendantInstance` does not receive the originating call surface (MCP vs CLI vs API) as a parameter. Attendant events emit `source: 'internal'` in this PR. Threading source through `handshake()` and `attend()` is a follow-up improvement.

2. **`escalation_expired` action type**: This action type (listed in the CP-T025 ticket) has no natural implementation path in `resolveInteractive()`, which only runs interactively when the operator explicitly runs `iranti resolve`. TTL expiry detection requires a scheduled pass. The recommended approach is to implement this in the Archivist's `processEscalations` path, where it can detect files past a configurable TTL during each maintenance cycle. This is out of scope for this PR; it can be added as a separate PR targeting the Archivist.

3. **`emitted_at` column**: The control plane's `staff_events` table migration (in the control plane repo) will need an `emitted_at TIMESTAMPTZ` column added to support latency measurement between `emit()` call time and SSE delivery time. This is a control plane migration concern, not an upstream package concern — the `emittedAt` field is already part of `StaffEvent` and `buildStaffEvent()` fills it correctly.

4. **Multi-instance `instanceId`**: If the Iranti server is ever run as multiple instances against the same database, an `instanceId` column on `staff_events` would allow routing events correctly. This is not required for the initial release and can be deferred.

5. **`context_loaded` injection point**: One Attendant injection point (`context_loaded`) was not identifiable from the compiled output. It is documented as a follow-up item in `diffs/attendant-changes.md`. All other required injection points are fully specified.
