# Iranti Control Plane — v0.4.0 Release Notes

**Status:** Release Candidate — 2026-03-21 (pending CP-T048 AC-11 clean-machine validation and GitHub Release tag)
**Previous release:** v0.3.0
**Epic:** CP-E015 — Iranti Desktop

---

## What's in v0.4.0

v0.4.0 is the "Iranti Desktop" release. The defining change is that opening the control plane now gives you a complete at-a-glance picture of your Iranti system — before clicking anything. Prior to this release, the app opened to the Health dashboard, which is a detailed diagnostic table. v0.4.0 introduces a true home page that behaves like a product landing screen, not an admin panel.

This release ships 3 tickets across Wave 10 (Phase 4), all PM-ACCEPTED on 2026-03-21.

User feedback that drove this release: *"you know how the Docker Desktop control panel looks? Why don't we have something like that?"*

## New Features

### Home Overview Dashboard

**CP-T068 — Wave 10**

A new `/overview` route is now the default landing page (replacing the `/health` redirect from all prior versions).

The overview dashboard is a 2-column card grid showing:

- **Alert Banner** — appears when any health check is degraded or critical; dismissible; links directly to the Health Dashboard
- **System Status Strip** — all health check results as mini color-coded pills (emerald = ok, amber = warn, rose = error); click anywhere to open Health Dashboard
- **KB Summary Card** — total facts in the knowledge base, facts written in the last 24 hours, and active agents this week; links to Metrics
- **Recent Activity Feed** — the last 8 Staff events with component badge (color-coded by component), humanized action type, agent ID, and relative timestamp; links to Staff Logs
- **Active Agents Card** — agents active in the last hour with status dot, total writes, and last-seen time; links to Agent Registry
- **Quick Actions Row** — four prominent action cards: Search KB, Run Diagnostics, Browse Memory, View Logs

The nav sidebar gains a **Home** item (⌂) as the first entry, above Memory.

Backend: a new `GET /api/control-plane/overview` endpoint aggregates all four data sources in a single `Promise.allSettled` call. Any source failure returns partial data — the endpoint never returns 500.

### Proactive Health Alert Toasts

**CP-T069 — Wave 10**

The control plane now tells you when something goes wrong while you are using any view — without requiring you to visit the Health Dashboard.

A lightweight toast notification system fires automatically when Iranti health degrades:

- **`warn` toast** (amber, 10s auto-dismiss): when health transitions from `healthy` to `degraded`
- **`error` toast** (rose, sticky): when health transitions to a critical error state
- **`info` toast** (emerald, 4s auto-dismiss): when health recovers back to `healthy`

Toasts appear in the bottom-right corner. They include a "View Health →" action link that navigates directly to the Health Dashboard. Each toast is deduplicated — the same alert will not repeat while the state remains unchanged. A health poller runs every 60 seconds in the background.

### Global Keyboard Navigation Shortcuts

**CP-T070 — Wave 10**

Power operators can now navigate to any view without touching the mouse using the `G + key` two-key shortcut pattern (the same pattern used in GitHub, Linear, and Notion).

Press **G**, then within 1.5 seconds press a letter key:

| Keys | Destination |
|------|-------------|
| `G → H` | Home (/overview) |
| `G → M` | Memory Explorer |
| `G → A` | Archive |
| `G → T` | Staff Activity |
| `G → L` | Logs |
| `G → I` | Instances |
| `G → D` | Health & Diagnostics |
| `G → X` | Metrics |
| `G → C` | Conflicts |
| `G → P` | Providers |
| `G → G` | Agent Registry |
| `G → S` | Getting Started |

A "go mode" indicator chip appears at the bottom-right while waiting for the second key. Shortcuts do not fire when focus is in a text input, textarea, or select element. Modifier key combinations (Ctrl+G, Cmd+G, etc.) are excluded.

All shortcuts are listed in the command palette's shortcuts panel (Cmd+K → type `?`).

## Breaking Changes

None. All v0.4.0 changes are additive. The only behavioral change is the default landing page: **`/` now redirects to `/overview` instead of `/health`**.

## Known Issues and Carry-Forward

### Release Candidate Status

v0.4.0 is currently at release candidate stage. A GitHub Release will be created when CP-T048 AC-11 (clean-machine installer validation) passes. The binary builds are complete and available in dist/. No GitHub Release tag has been pushed and no formal release exists on the repository at this time.

### CP-T048 AC-11 — Platform installer clean-machine validation

The v0.3.0 gate (CP-T048 AC-11) remains open — this does not block v0.4.0 since it is a v0.3.0 artefact. The installers are built; clean-machine testing is the outstanding step. CP-T075 is the Phase 5 ticket to formally run this validation.

### Go mode chip z-index coordination

The go mode chip (CP-T070) uses `z-index: 1050`. Toasts (CP-T069) use `z-index: 1100`. If both appear simultaneously, toasts render above the go mode chip, which is correct. No conflict.

### Overview data freshness

The overview dashboard polls every 30 seconds. On instances with very high Staff event frequency, the "Recent Activity" feed may lag by up to 30 seconds. Future work may add SSE wiring to the activity feed for real-time updates.
