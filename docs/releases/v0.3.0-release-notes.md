# Iranti Control Plane — v0.3.0 Release Notes

**Status:** Release Candidate — 2026-03-21
**Gate:** CP-T048 AC-11 (clean-machine installer validation) is the sole remaining condition for final release
**Previous release:** v0.2.0-beta (2026-03-20)

## What's in v0.3.0

v0.3.0 is a major capability release. It ships all Phase 3 work across 9 delivery waves, completing the "Knowledge Base Power Features" epic and bringing the control plane from a passive inspection tool to a full knowledge management and diagnostic surface.

---

## New Features

### KB Full-Text / Semantic Search

**CP-T066 — Wave 9**

A global cross-KB search input now sits at the top of the Memory Explorer. Operators can search across the entire knowledge base using Iranti's hybrid lexical + vector search — answering questions like "which entities mention Project Iris?" or "what do agents know about onboarding?" without knowing entity type or ID upfront.

- Results ranked by combined relevance score (shown as a percentage bar)
- Per-result: entity link, fact key, value summary, confidence, source
- If vector search is unavailable, falls back to lexical-only with a clear note
- Scope error (namespace-scoped key) surfaced with a clear explanation
- Requires a global-scope Iranti API key (`kb:read` global)

### Entity Type Browser

**CP-T067 — Wave 9**

The Memory Explorer's default landing state is now an Entity Type Browser — a card grid of all distinct entity types in the KB, each showing fact count and last-updated time. Clicking "Browse →" filters to that type. This replaces the previous empty-table initial state.

- Ideal for operators exploring an unfamiliar Iranti instance
- URL-shareable: `?entityType=agent` links directly to a filtered browse
- Empty KB shows a helpful onboarding message linking to Staff Logs
- 2-minute client-side cache (entity types change infrequently)

### Entity Alias Management

**CP-T061/CP-T065 — Wave 8/9**

A fourth "Aliases" tab in the Entity Detail page shows all registered alias tokens for an entity. Aliases are human-readable lookup shorthands (e.g., `alice` for `user/alice-doe`) — not entity cross-references.

- Each alias shows the token (monospace), source (manual/query), confidence bar, and registration age
- Operators can register new aliases with a single-field form (canonical entity derived from the current page)
- POST body matches the real Iranti v0.2.15 alias API shape exactly

### Interactive Diagnostics Panel

**CP-T059 — Wave 6**

A "Run Diagnostics" button in the Health Dashboard actively probes 7 system checks:

| Check | What it tests |
|---|---|
| `iranti_connectivity` | Iranti `/health` reachability |
| `iranti_auth` | API key scope (via `/kb/search?query=test`) |
| `db_connectivity` | Control plane Postgres `SELECT 1` |
| `vector_backend` | qdrant/chroma/pgvector reachability |
| `ingest_roundtrip` | Write → read → delete a probe fact |
| `attend_check` | Attendant classifier parse success |
| `vector_search_check` | Vector score > 0 in search results |

Results show pass/warn/fail with actionable fix hints and per-check duration. Previous run is shown on page load (in-memory cache). Also reachable from the command palette (Cmd+K → "Run Diagnostics").

### Metrics Dashboard

**CP-T060 — Wave 7**

A new `/metrics` view with SVG-native charts (no external chart library):

- KB growth chart: new facts, archived facts, cumulative total over 7d/30d
- Agent activity chart: per-agent write/rejection/escalation stacked bars, top 5 + "Other" grouping
- 4 summary stat cards: total facts, facts last 24h, facts last 7d, active agents last 7d
- Period toggle (7d/30d) re-fetches via separate query keys
- Graceful degradation if `staff_events` table is absent

### Contributors Panel (WhoKnows)

**CP-T057 — Wave 5**

A "Contributors" tab in the Entity Detail page lists every agent that has written facts to that entity, ranked by write count, with last-contribution timestamp and a link to the agent's Agent Registry profile.

### Temporal asOf Query

**CP-T056 — Wave 5**

An "As Of" date picker in the Temporal History page header lets operators query what value a fact held at any point in the past. The matching historical interval is highlighted in the timeline. Uses a direct-DB query (knowledge_base first, archive fallback) for accuracy.

### Agent Registry

**CP-T051 — Wave 4**

A read-only view of all registered agents at `/agents` — last seen, active status, write volume, rejection rate, escalation count. Per-agent detail panel with capability list and write history.

### Capability Health Cards

**CP-T052 — Wave 4**

Three new Health Dashboard cards — Memory Decay configuration, Vector Backend reachability, and Attendant health signal — without affecting the overall health indicator.

### Conflict History in Memory/Archive

**CP-T053 — Wave 4**

Expanded fact rows now show a ConflictLog timeline, field-level stability, last-accessed data, and clear "Written by" / "Source" labels. Archive rows show Archivist decision history.

### Archivist Decision Transparency and Flagging

**CP-T049 — Wave 2**

Archive Explorer now shows per-fact Archivist event timelines. Operators can flag suspicious facts for review and restore superseded values. Flagged queue at `/archive?flagged=true`.

### Staff Logs

**CP-T050 — Wave 1**

Persistent, queryable Staff event history at `/logs`. Filter by component, date range, severity, agent, and event type. Export as JSONL or CSV. Paginated, fast.

### Entity Relationship Graph

**CP-T032 — Phase 2**

Interactive radial graph in the Entity Detail Relationships tab — depth 1 or 2, click nodes to navigate.

- **`__diagnostics__` filter (CP-T059 AC-9):** Memory Explorer default browse filters out `entityType: '__diagnostics__'` probe facts written by the ingest_roundtrip diagnostic check.

## Known Issues and Open Flags

### B6 — Ingest contamination risk

B6 (ingest contamination): **FIXED upstream in v0.2.16**. `iranti_ingest` prose extraction is now benchmark-confirmed working. The `iranti_write` workaround documented here is no longer required for operators running Iranti v0.2.16+.

The diagnostic probe fact filter (`entityType: '__diagnostics__'`) in the Memory Explorer UI remains in place as a hygiene measure and is unaffected by the B6 fix.

### B11 — Attend classifier

Iranti v0.2.13 partially fixed the attend classifier parse failure (`classification_parse_failed_default_false`), and v0.2.14 confirms that `user/main` entity recovery now works correctly without `entityHints`. The `user/main` noise previously flagged as an open defect is **resolved upstream**. The `attend_check` diagnostic in the Interactive Diagnostics Panel probes for remaining parse failures and warns if any are observed. Operators on v0.2.12 or earlier should upgrade.

### B12 — Transaction timeout on LLM-arbitrated writes (upstream, HIGH PRIORITY)

**New — confirmed 2026-03-21.** When Iranti resolves a fact conflict via LLM arbitration, the database transaction may time out before the LLM round-trip completes, causing the write to be rolled back silently. Non-conflicting writes are unaffected. Multi-agent workflows where two agents write to the same entity/key are most exposed. **Workaround:** Use `iranti_write` with an explicit resolved value to manually resolve conflicts if you observe accumulating low-confidence conflicting facts. See `docs/coordination/upstream-bug-flags-2026-03-21.md` B12 for full details.

### CP-T025 — Upstream PR pending

The upstream PR to add native Staff event emission for Attendant and Resolutionist components has been spec'd and diff files produced (`docs/specs/cp-t025-diffs/`). The PR has not been submitted — awaiting upstream repo path confirmation. Until merged, the Staff Activity Stream and Staff Logs show Librarian and Archivist events only.
